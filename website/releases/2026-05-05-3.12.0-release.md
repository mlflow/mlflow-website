MLflow 3.12.0 is a release focused on improving our LLM observability workflows, making tracing more accessible, feature-rich, and performant.

## 🖼️ Multimodal Tracing

<div style={{ display: "flex", gap: "1rem", alignItems: "flex-start", flexWrap: "wrap" }}>
  <img alt="Trace with image" src="/img/releases/3.12.0/trace-image-content.png" style={{ maxWidth: "48%", height: "auto", flex: "1 1 320px", minWidth: 0 }} />
  <img alt="Trace with audio" src="/img/releases/3.12.0/trace-audio-player.png" style={{ maxWidth: "48%", height: "auto", flex: "1 1 320px", minWidth: 0 }} />
</div>

Users can now store multimodal content in tracing spans as artifact attachments instead of inline binary data.  We've also patched the UI to support the new mlflow-attachment:// style URI, with rich rendering available for PDFs, audio, and images.

## 🤖 Codex, Gemini, Qwen coding agent tracing support

![Codex Tracing](/img/releases/3.12.0/codex-tracing.png)

Similar to our Claude Code tracing integration, we've now added support for the Codex, Gemini, and Qwen coding agent platforms as well!

## 🛡️ Gateway guardrails

You can now set guardrails on your gateway endpoints to prevent unsafe or non-compliant model inputs and outputs. Try it out in the MLflow UI!

## ⚡ Trace table pagination

<video controls autoplay muted loop playsinline width="100%">
  <source
    src={require("/img/releases/3.12.0/trace-pagination.mp4").default}
    type="video/mp4"
  />
</video>

The traces tab is now paginated, rather than fetching all traces up to a limit of 1000. This improves initial load time, and makes the page feel more responsive overall.
