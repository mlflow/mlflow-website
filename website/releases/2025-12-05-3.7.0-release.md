MLflow 3.7.0 includes several major features and improvements for GenAI Observability, Evaluation, and Prompt Management.

### Major Features

- üìù **Experiment Prompts UI**: New prompts functionality in the experiment UI allows you to manage and search prompts directly within experiments, with support for filter strings and prompt version search in traces. ([#19156](https://github.com/mlflow/mlflow/pull/19156), [#18919](https://github.com/mlflow/mlflow/pull/18919), [#18906](https://github.com/mlflow/mlflow/pull/18906), [@TomeHirata](https://github.com/TomeHirata))
- üí¨ **Multi-turn Evaluation Support**: Enhanced `mlflow.genai.evaluate` now supports multi-turn conversations, enabling comprehensive assessment of conversational AI applications with DataFrame and list inputs. ([#18971](https://github.com/mlflow/mlflow/pull/18971), [@AveshCSingh](https://github.com/AveshCSingh))
- ‚öñÔ∏è **Trace Comparison**: New side-by-side comparison view in the Traces UI allows you to analyze and debug LLM application behavior across different runs, making it easier to identify regressions and improvements. ([#17138](https://github.com/mlflow/mlflow/pull/17138), [@joelrobin18](https://github.com/joelrobin18))
- üåê **Gemini TypeScript SDK**: Auto-tracing support for Google's Gemini in TypeScript, expanding MLflow's observability capabilities for JavaScript/TypeScript AI applications. ([#18207](https://github.com/mlflow/mlflow/pull/18207), [@joelrobin18](https://github.com/joelrobin18))
- üéØ **Structured Outputs in Judges**: The `make_judge` API now supports structured outputs, enabling more precise and programmatically consumable evaluation results. ([#18529](https://github.com/mlflow/mlflow/pull/18529), [@TomeHirata](https://github.com/TomeHirata))
- üîó **VoltAgent Tracing**: Added auto-tracing support for VoltAgent, extending MLflow's observability to this AI agent framework. ([#19041](https://github.com/mlflow/mlflow/pull/19041), [@joelrobin18](https://github.com/joelrobin18))

### Breaking Changes

- [Tracking] SQLite is now the default backend for the MLflow Tracking server. ([#18497](https://github.com/mlflow/mlflow/pull/18497), [@harupy](https://github.com/harupy))
- [Models] Remove deprecated `diviner` flavor ([#18808](https://github.com/mlflow/mlflow/pull/18808), [@copilot-swe-agent](https://github.com/copilot-swe-agent))
- [Models] Remove deprecated `promptflow` flavor ([#18805](https://github.com/mlflow/mlflow/pull/18805), [@copilot-swe-agent](https://github.com/copilot-swe-agent))
